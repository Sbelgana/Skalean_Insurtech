## Objectif Global du Sprint

Construire l'application **Assurflow Carrier Portal** : interface dediee aux compagnies d'assurance (acteur 1/6 ecosystem v3.0) pour gerer claims, valider paiements multi-niveau, designer experts, generer reports ACAPS, suivre stats partenaires brokers/garages/experts.

**Pourquoi sprint critique** : les carriers sont les **seuls clients paying recurrents** dans le modele Assurflow v3.0. Sans Carrier Portal :
- Pitch Demo Day 30 juin = pas de revenue source claire
- Lettre d'intent co-developpement impossible
- Differentiation vs Courtizen perdue (Courtizen attaque carriers via API uniquement)

A la sortie de ce sprint :
- App `web-carrier-portal` Next.js 15 desktop (port 3008 dev / `app.assurflow.com/carrier` prod)
- 15 pages applicatives : login + MFA + dashboard + claims + claim detail + payment approval + experts pool + experts designate + experts performance + partners stats + brokers manage + garages manage + ACAPS reports + fraud alerts + parametres
- 6 roles UI : carrier_admin / carrier_claims_manager / carrier_finance / carrier_compliance / carrier_expert_manager / carrier_partner_manager
- Backend complete : 4 entites DB (carrier_dashboards_config + carrier_compliance_reports + carrier_payment_approvals + carrier_partners_stats)
- Workflow paiement multi-niveau : 4 paliers (L1 < 5000 / L2 5000-20000 / L3 20000-100000 / L4 > 100000 MAD)
- Workflow designation experts : carrier designe expert agree ACAPS pool prive
- ACAPS reports automatises trimestriels + annuels + ad-hoc
- Fraud detection alerts dashboard
- Stats partenaires : performance brokers (volume + pipeline) + garages (volume + delais) + experts (rapports + qualite)
- I18n fr/ar-MA/ar

## Differentiation strategique

**Constat marche MA** :
- 8 carriers majeurs : Wafa Assurance + Sanlam + AXA + RMA + Saham + Atlanta + MATU + Sanad
- Pain points carriers identifies (analyse strategique v2.0) :
  1. Workflow paiement manuel papier (delais 30-180 jours)
  2. Designation experts par fax/email manual (lenteur + tracability nulle)
  3. ACAPS reports trimestriels generes manuellement (semaines effort)
  4. Pas de visibility temps reel sur claims chain (broker -> garage -> expert -> paiement)
  5. Fraud detection basique (humain seulement)
  6. Stats partenaires fragmentees (Excel manuel)
  7. Cas concret : 400k MAD impayes MATU au garage de Saad (pain pour garages + reputation carrier)

**Assurflow Carrier Portal resout 7 pain points** :
1. Workflow paiement digital multi-niveau (7 jours vs 30-180)
2. Designation expert 1-click depuis Carrier Portal
3. ACAPS reports auto-generes (trimestriel + annuel + ad-hoc)
4. Dashboard temps reel sinistre chain visible
5. Fraud detection avec alertes ML (Sprint 30+ enrichi)
6. Stats partenaires unifiees + temps reel
7. Transparence + delais respectes = reputation carrier amelioree

**Effet marketing** : "Premier portail unifie pour carriers MA -- compliance ACAPS automatisee + workflow claims temps reel" = pitch lettre d'intent Demo Day 30 juin.

import Decimal from 'decimal.js';

@Injectable()
export class CarrierPaymentApprovalsService {
  constructor(
    @InjectRepository(CarrierPaymentApproval)
    private approvalsRepo: Repository<CarrierPaymentApproval>,
    private notificationsService: NotificationsService,
    private payService: PaymentGatewayService,
    private auditLogService: AuditLogService,
    private kafkaProducer: KafkaProducer,
    private rbacService: RbacService,
  ) {}
  
  /**
   * Determine required approval level selon montant.
   * Rules v3.0 (configurable env vars per tenant) :
   *   L1 < 5000 MAD
   *   L2 5000-20000 MAD
   *   L3 20000-100000 MAD
   *   L4 > 100000 MAD
   */
  determineRequiredLevel(amountMad: number): 1 | 2 | 3 | 4 {
    const amount = new Decimal(amountMad);
    
    if (amount.lessThan(5000)) return 1;
    if (amount.lessThan(20000)) return 2;
    if (amount.lessThan(100000)) return 3;
    return 4;
  }
  
  /**
   * Create approval request and notify L1 approver.
   */
  async createApproval(input: CreateApprovalInput): Promise<CarrierPaymentApproval> {
    const level = this.determineRequiredLevel(input.amountMad);
    
    const approval = await this.approvalsRepo.save({
      tenant_id: input.tenantId,
      sinistre_id: input.sinistreId,
      devis_id: input.devisId,
      expert_report_id: input.expertReportId,
      payee_tenant_id: input.payeeTenantId,
      payee_type: input.payeeType,
      amount_mad: new Decimal(input.amountMad).toFixed(2),
      required_level: level,
      status: 'pending_l1',
      expected_payment_date: input.expectedPaymentDate,
    });
    
    // Notify L1 approver (carrier_claims_manager)
    await this.notifyLevelApprovers(approval.id, 1);
    
    // Emit event
    await this.kafkaProducer.emit('insurtech.events.carrier.approval.created', {
      approvalId: approval.id,
      amountMad: input.amountMad,
      requiredLevel: level,
    });
    
    // Audit log
    await this.auditLogService.log({
      entity_type: 'carrier_payment_approval',
      entity_id: approval.id,
      action: 'created',
      actor_user_id: input.actorUserId,
      details: { amount: input.amountMad, level },
    });
    
    return approval;
  }
  
  /**
   * Approve at current level.
   * Auto-progress to next level OR mark fully_approved if level matches required.
   */
  async approve(
    approvalId: string,
    approverUserId: string,
    comments?: string
  ): Promise<void> {
    const approval = await this.approvalsRepo.findOneOrFail({ where: { id: approvalId } });
    
    // Determine current level
    const currentLevel = this.getCurrentApprovalLevel(approval.status);
    if (currentLevel === null) {
      throw new BadRequestException(`Approval ${approvalId} is not in pending state (status: ${approval.status})`);
    }
    
    // Verify approver has required permission
    await this.verifyApproverPermissions(approverUserId, currentLevel);
    
    // Update current level
    const levelField = `l${currentLevel}_approver_user_id` as keyof CarrierPaymentApproval;
    const dateField = `l${currentLevel}_approved_at` as keyof CarrierPaymentApproval;
    const commentField = `l${currentLevel}_comments` as keyof CarrierPaymentApproval;
    
    const updates: Partial<CarrierPaymentApproval> = {
      [levelField]: approverUserId,
      [dateField]: new Date(),
      [commentField]: comments,
    };
    
    // Determine next status
    if (currentLevel === approval.required_level) {
      // Fully approved -> trigger payment
      updates.status = 'fully_approved';
      
      await this.approvalsRepo.update(approvalId, updates);
      
      // Emit event
      await this.kafkaProducer.emit('insurtech.events.carrier.approval.fully_approved', {
        approvalId,
        finalApproverUserId: approverUserId,
        level: currentLevel,
      });
      
      // Trigger payment via @insurtech/pay (Sprint 11)
      await this.triggerPayment(approvalId);
    } else {
      // Move to next level
      const nextLevel = currentLevel + 1;
      updates.status = `pending_l${nextLevel}` as any;
      
      await this.approvalsRepo.update(approvalId, updates);
      
      // Emit event
      await this.kafkaProducer.emit(`insurtech.events.carrier.approval.approved_l${currentLevel}`, {
        approvalId,
        approverUserId,
        nextLevel,
      });
      
      // Notify next level approvers
      await this.notifyLevelApprovers(approvalId, nextLevel);
    }
    
    // Audit log
    await this.auditLogService.log({
      entity_type: 'carrier_payment_approval',
      entity_id: approvalId,
      action: `approved_l${currentLevel}`,
      actor_user_id: approverUserId,
      details: { level: currentLevel, comments },
    });
  }
  
  /**
   * Get current pending level from status.
   */
  getCurrentApprovalLevel(status: string): 1 | 2 | 3 | 4 | null {
    if (status === 'pending_l1') return 1;
    if (status === 'pending_l2') return 2;
    if (status === 'pending_l3') return 3;
    if (status === 'pending_l4') return 4;
    return null;
  }
  
  /**
   * Verify approver has required permission for given level.
   */
  async verifyApproverPermissions(approverUserId: string, level: 1 | 2 | 3 | 4): Promise<void> {
    const permission = `carrier.payment.approve_level${level}` as PermissionValue;
    
    const hasPermission = await this.rbacService.userHasPermission(approverUserId, permission);
    if (!hasPermission) {
      throw new ForbiddenException(
        `User ${approverUserId} does not have permission ${permission}`
      );
    }
  }
  
  /**
   * Trigger payment after fully_approved.
   */
  private async triggerPayment(approvalId: string): Promise<void> {
    const approval = await this.approvalsRepo.findOneOrFail({ where: { id: approvalId } });
    
    // Create payment via @insurtech/pay (Sprint 11)
    const payTransaction = await this.payService.createPayment({
      from_tenant_id: approval.tenant_id,
      to_tenant_id: approval.payee_tenant_id,
      amount_mad: approval.amount_mad,
      reference: `Sinistre ${approval.sinistre_id} -- Approval ${approval.id}`,
      gateway: 'cmi',  // default carrier -> garage
      metadata: { approvalId: approval.id, sinistreId: approval.sinistre_id },
    });
    
    // Update approval
    await this.approvalsRepo.update(approvalId, {
      payment_transaction_id: payTransaction.id,
      payment_initiated_at: new Date(),
      status: 'paid',
    });
    
    // Calculate delay
    if (approval.expected_payment_date) {
      const delayDays = differenceInDays(new Date(), approval.expected_payment_date);
      await this.approvalsRepo.update(approvalId, { delay_days: Math.max(0, delayDays) });
    }
    
    // Emit final event
    await this.kafkaProducer.emit('insurtech.events.carrier.approval.paid', {
      approvalId,
      payTransactionId: payTransaction.id,
    });
  }
  
  /**
   * Reject at any level with reason.
   */
  async reject(
    approvalId: string,
    approverUserId: string,
    reason: string,
    level: 1 | 2 | 3 | 4
  ): Promise<void> {

  }
}
